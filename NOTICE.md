## Synthetic Data

This repository ships **only synthetic data**. No customer information,
production records, or other real-world data are included anywhere in
the codebase, the notebooks, the bundle resources, or the volumes
created at deployment time.

The synthetic data is generated at deploy time by the notebooks in
`fins-aml-data-bundle/notebooks/`. The generation pipeline produces:

| Output | Generated by | Storage |
|---|---|---|
| Customers, accounts, transactions, alerts, cases, SAR filings, audit logs | `01_aml_data_generation.py` — combination of [Faker](https://faker.readthedocs.io/) for entity attributes, statistical distributions for amounts/dates/risk scores, and a Databricks-hosted foundation model (called via `ai_query()`) for transaction memo prose | Delta tables under `<catalog>.<schema>.*` |
| Watchlist screening results | `02_aml_watchlist_screening.py` — fuzzy-matching the synthetic customer set against a synthetic watchlist | Delta tables under `<catalog>.<schema>.*` |
| Graph nodes and edges | `03_aml_graph_model.py` — derived deterministically from the relational tables above | `<catalog>.<schema>.graph_nodes`, `<catalog>.<schema>.graph_edges` |
| SAR narratives, case notes, EDD memos, adverse media reports, customer correspondence | `04_aml_knowledge_base.py` — generated by a Databricks-hosted foundation model (via `ai_query()`) prompted with structured fields drawn from the tables above | UC volume `/Volumes/<catalog>/<schema>/knowledge_base/` |

All generation code is in this repository under the same LICENSE as the
rest of the project. The pipeline is fully repeatable — partners who
deploy the bundle will get a fresh, independent synthetic dataset.

External datasets (e.g. real watchlists, real customer correspondence)
are not used and not required.
